# 컨텍스트

## 배경
PRD v2.2 기반 MVP 1차 개발. 핵심 플로우: 링크 생성 → 카톡 공유 → 연동/입력 → 실시간 현황 → 시간 확정 → 캘린더 자동 추가

## 주요 결정사항
1. **인증**: 소셜 로그인(카카오/구글) + 게스트(비로그인) 하이브리드
2. **게스트→회원 머지**: 필수 (재방문 자동 반영이 핵심 가치)
3. **캘린더 데이터**: 서버에 이벤트 내용 저장 안 함, 가용/불가용 여부만 추출
4. **MVP 캘린더**: Google Calendar 우선, Apple CalDAV는 Phase 2
5. **에브리타임**: Gemini Vision으로 블록 감지 (과목명 인식 불필요)

## 제약
- Supabase Free Tier (500MB DB, 50K MAU)
- Google Calendar API 무료 (일 1M 요청)
- Gemini API Free (15 RPM, 일 1,500 요청)

## 리스크
- 게스트 세션 유실 시 데이터 복구 불가 → 로그인 유도 UX 필요
- OAuth 토큰 보안 → RLS + 암호화 고려
