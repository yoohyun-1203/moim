# Research 템플릿

## 기술 비교 보고서

```markdown
## 조사 보고서: [주제]

### 조사 목적
- [무엇을 결정하기 위한 조사인지]

### 현재 상황
- 프로젝트: [프로젝트명]
- 기존 스택: [관련 기술]
- 제약 조건: [성능/비용/팀 규모 등]

### 선택지 비교

| 기준 | A | B | C |
|------|---|---|---|
| 성능 | | | |
| 학습 곡선 | | | |
| 커뮤니티/생태계 | | | |
| 현재 스택 호환성 | | | |
| 유지보수 비용 | | | |
| 라이선스 | | | |

### 추천
- **1순위**: [선택지] — [핵심 근거 (출처)]
- **대안**: [선택지] — [어떤 경우에 더 나은지]

### 리스크
- [주의해야 할 점]

### 다음 단계
- [조사 결과를 바탕으로 할 일]
```

## Quick-scan 요약

```markdown
## [주제] 빠른 조사

**결론**: [한 줄 추천]

| 선택지 | 장점 | 단점 | 적합도 |
|--------|------|------|--------|
| A | | | ★★★ |
| B | | | ★★☆ |

**근거**: [핵심 이유 2-3줄]
**다음 단계**: [바로 할 일]
```

## 패턴/방법론 조사

```markdown
## [패턴명] 조사 보고서

### 개요
- [패턴이 해결하는 문제]

### 적용 방법
1. [단계별 설명]

### 우리 프로젝트에 적용 시
- 장점: [구체적 이점]
- 비용: [도입 비용/리스크]
- 영향 범위: [변경이 필요한 부분]

### 레퍼런스
- [출처 1]
- [출처 2]
```
