# Research 에러 플레이북

## 정보 부족

**증상**: 웹 검색 결과가 불충분하거나 오래된 정보만 나옴.
**대응**:
1. 검색 키워드 변경 (동의어, 영어 키워드 추가)
2. GitHub 리포지토리 직접 확인
3. 공식 문서 URL 직접 접근
4. 사용자에게 추가 컨텍스트 요청

## 선택지 과다

**증상**: 비교할 선택지가 5개 이상으로 수렴하지 않음.
**대응**:
1. 프로젝트 제약 조건으로 1차 필터링
2. 커뮤니티 크기/활성도로 2차 필터링
3. 최종 2-3개로 좁혀서 상세 비교

## 편향된 정보

**증상**: 특정 기술의 마케팅 자료만 나옴.
**대응**:
1. GitHub Issues에서 실제 사용자 피드백 확인
2. "alternatives to X", "X vs Y" 검색
3. 단점/한계를 명시적으로 검색

## 조사 범위 확장

**증상**: 조사하다 보니 범위가 계속 넓어짐.
**대응**:
1. 원래 조사 목적 재확인
2. 사용자에게 범위 조정 확인
3. 현재 범위에서 결론 가능하면 일단 보고
