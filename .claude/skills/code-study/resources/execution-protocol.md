# code-study 실행 프로토콜

## Step 0: 레벨 결정

| 조건 | 레벨 | 범위 |
|------|------|------|
| 기본 (아무 선언 없음) | **S** | 5단계 전체 |
| "간단히" 선언 | **A** | What → Why → 개념 (3단계) |
| "바로 넘어가자" 선언 | **B** | What 체크리스트만 |

- 완전 스킵 불가 — 최소 What 체크리스트 확인은 필수

## Step 1: 변경 전체 스캔 → 체크리스트 생성

1. `git diff`로 변경된 파일 목록 수집
2. 파일별 변경 내용을 의미 단위로 분류:
   - 새 함수/블록 추가
   - 기존 로직 수정
   - 패턴/상수 추가
   - 구조 변경 (파일 분리, 재구조화 등)
3. 체크리스트 형식으로 출력:

```
📖 코드 스터디 시간입니다 (레벨 S):

오늘 변경된 코드를 정리했습니다:

[파일명1] (수정)
- [ ] 항목1: {무엇이 변경되었는지} (라인 범위)
- [ ] 항목2: {무엇이 변경되었는지} (라인 범위)

[파일명2] (신규)
- [ ] 항목3: {무엇이 추가되었는지}
- [ ] 항목4: {무엇이 추가되었는지}

하나씩 확인합니다. 첫 번째부터:
→ {항목1}이 뭘 하는 건지 설명해봐
```

4. 사용자가 답변하면 이해 검증 루프 적용
5. 정확하면 `[x]` 체크 → 다음 항목

## Step 2: Why (설계 의도) 질문

체크리스트 항목 중 설계 결정이 있는 것들에 대해:

- "왜 이렇게 구현했어?"
- "다른 방법은 없었어? 있었다면 왜 이걸 골랐어?"
- 트레이드오프가 있는 경우: "이 방식의 장단점은?"

**주의**: 모든 항목에 Why를 물을 필요 없음. 단순 추가(패턴 추가 등)는 스킵.
설계 결정이 있는 항목만 선별.

## Step 3: 개념 추출 → 질문

1. 변경 코드에서 사용된 **기반 개념/도구** 목록을 추출:

```
💡 이 코드에서 사용된 개념들:
- {개념1}: {코드에서 사용된 위치}
- {개념2}: {코드에서 사용된 위치}
- {개념3}: {코드에서 사용된 위치}
```

2. 각 개념별로 질문:
   - "이 개념이 뭔지 알아?"
   - 안다면: "구체적으로 설명해봐"
   - 모른다면: 함께 설명 → **반드시 확인 질문**

### 개념 추출 기준

| 범주 | 예시 |
|------|------|
| **언어/쉘 기능** | exit code, set -euo pipefail, 변수 치환(${VAR:-default}) |
| **도구/명령어** | jq, grep -qiE, sed, stat, wc |
| **정규식** | `^\\.env$`, `\.\s*$`, `-qiE` 플래그 의미 |
| **패턴/설계** | early return vs 플래그 패턴, guard clause |
| **시스템 개념** | stdin/stdout/stderr, 프로세스 종료 코드, 락 파일 |
| **프레임워크 특화** | Claude Code 훅 이벤트, exit 2의 의미 |

## Step 4: 연결 (개념 ↔ 코드)

개념을 알더라도, **이 코드에서의 구체적 역할**을 연결할 수 있는지 확인:

- "exit 2가 뭔지는 알겠어. 그러면 이 훅에서 exit 2를 쓰면 Claude Code는 어떻게 반응해?"
- "grep -qiE의 각 플래그가 뭔지는 알겠어. 여기서 -i를 빼면 어떻게 달라져?"

## Step 5: 응용 (변형/확장)

구체적 시나리오를 제시하여 응용력 확인:

- "새로운 차단 패턴을 추가하려면?"
- "이 로직에 예외를 만들려면?"
- "이 훅이 실패하면 전체 흐름에 어떤 영향?"

## 이해 검증 루프 (모든 Step에 공통 적용)

```
┌─ 질문
│
├─ 답변 평가
│   ├─ ✅ 정확     → 체크 [x] → 다음
│   ├─ ⚠️ 부분이해  → 힌트 제공 → 같은 질문 재시도
│   ├─ ❌ 오해     → "그 부분이 좀 달라" + 힌트 → 재시도
│   └─ ❓ 모름     → 함께 읽기/설명
│                      └─ 확인 질문 (필수!)
│                          ├─ 정확 → 다음
│                          └─ 부정확 → 한 번 더 설명 → 넘어감
└─ 3회 힌트 후에도 미해결 → 전체 설명 + 확인 질문 → 넘어감
```

**절대 규칙**:
- 설명만 하고 끝내지 않는다. 반드시 확인 질문을 던진다.
- "내가 방금 설명한 거 한 줄로 정리해볼래?" 또는 "그러면 {구체적 질문}?"
