# Document 에러 플레이북

## 코드와 문서 불일치

**증상**: 기존 문서 내용이 현재 코드와 다름.
**대응**:
1. git log로 코드 변경 시점 확인
2. 코드를 진실로 삼아 문서 갱신
3. 변경 이력에 "코드 기반 갱신" 명시

## 문서 스타일 불일치

**증상**: 프로젝트 내 문서 스타일이 제각각.
**대응**:
1. 가장 최근/가장 많은 문서의 스타일을 기준으로 삼기
2. 현재 작업 범위의 문서만 통일
3. 전체 통일은 별도 태스크로 제안

## 과도한 문서화 요청

**증상**: 모든 함수에 docstring, 모든 파일에 주석 요청.
**대응**:
1. public API만 문서화하도록 범위 조정 제안
2. 자명한 코드는 문서화 불필요함을 설명
3. 복잡한 비즈니스 로직, 비직관적 결정에만 집중

## 대상 코드 접근 불가

**증상**: 문서화 대상 코드를 찾을 수 없음.
**대응**:
1. 파일 경로 재확인
2. 사용자에게 정확한 대상 확인
3. glob/grep으로 관련 코드 탐색
