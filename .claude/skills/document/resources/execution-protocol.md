# Document 실행 프로토콜

## 1단계: 문서화 범위 파악

```
체크리스트:
□ 문서 유형 결정 (API / 아키텍처 / 가이드 / 코드 / 변경)
□ 대상 코드/모듈 확인
□ 기존 문서 유무 확인 (신규 생성 vs 갱신)
□ 대상 독자 확인 (개발자 / 사용자 / 팀원)
□ 프로젝트 문서 스타일 확인
```

## 2단계: 코드 분석

1. 대상 코드 읽기
   - 함수 시그니처, 타입, 파라미터
   - 주석, docstring, JSDoc
   - export/public API 식별
2. 의존성 추적
   - import/require 관계
   - 데이터 흐름
3. 테스트 코드 확인
   - 사용 패턴, 엣지 케이스
   - 기대 동작

## 3단계: 문서 작성

### 신규 생성
1. `doc-templates.md`에서 적절한 템플릿 선택
2. 코드 분석 결과로 내용 채우기
3. 실행 가능한 예시 코드 포함
4. 프로젝트 한국어 문서 기준 적용

### 기존 갱신
1. 기존 문서 전체 읽기
2. 변경된 코드와 문서 비교
3. 변경 부분만 업데이트 (surgical diff)
4. 기존 톤/스타일 유지

## 4단계: 검증

```
검증 체크리스트:
□ 문서 내용이 실제 코드와 일치하는가?
□ 예시 코드가 실제 동작 가능한가?
□ 파일 경로, 링크가 유효한가?
□ 한국어 문서 기준을 준수하는가? (식별자는 영어)
□ 불필요한 중복이 없는가?
```
