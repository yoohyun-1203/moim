# Context-Builder 에러 플레이북

## 패키지 매니저 파일 없음

**증상**: package.json, pyproject.toml 등이 없어 기술 스택 감지 불가.
**대응**:
1. 소스 코드의 import 문에서 기술 스택 추론
2. Dockerfile이 있으면 베이스 이미지에서 추론
3. 불확실한 항목은 `[확인 필요]`로 표시
4. 사용자에게 기술 스택 직접 확인

## 모노레포 / 멀티 프로젝트

**증상**: 루트에 여러 프로젝트가 있어 구조가 복잡.
**대응**:
1. 각 서브 프로젝트를 개별 섹션으로 분리
2. 공유 라이브러리/모듈 별도 식별
3. 프로젝트 간 의존성 관계 문서화

## 기존 CLAUDE.md와 충돌

**증상**: 자동 감지 결과가 기존 CLAUDE.md 내용과 다름.
**대응**:
1. 기존 내용을 진실로 간주 (사용자가 의도적으로 작성)
2. 차이점만 별도로 보고
3. 사용자에게 갱신 여부 확인

## 대규모 코드베이스

**증상**: 파일이 너무 많아 전체 스캔이 비효율적.
**대응**:
1. 루트 설정 파일만 먼저 분석
2. src/ 또는 app/ 최상위 디렉토리 구조만 파악
3. 핵심 모듈부터 점진적으로 문서화
