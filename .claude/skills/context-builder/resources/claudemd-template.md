# CLAUDE.md 템플릿

프로젝트 스캔 결과를 기반으로 CLAUDE.md를 생성할 때 사용하는 템플릿.

```markdown
# [프로젝트명] — Claude Code 프로젝트 설정

[프로젝트 한 줄 설명]

## 사용자 환경

- 언어: [ko / en]
- 호칭: [사용자 호칭]
- 문체: [문체 설명]
- 시간대: [Asia/Seoul 등]
- 보고 형식: What, Why, Result

## 기술 스택

| 영역 | 기술 | 버전 |
|------|------|------|
| 런타임 | [Node.js / Python 등] | [버전] |
| 프레임워크 | [Next.js / FastAPI 등] | [버전] |
| DB | [PostgreSQL 등] | [버전] |
| ORM | [Prisma / SQLAlchemy 등] | [버전] |

## 프로젝트 구조

\`\`\`
[디렉토리 트리 — 주요 디렉토리만]
\`\`\`

## 핵심 원칙

1. 간결 우선: 요청 범위에 맞는 최소 구현
2. 외과적 변경: 관련 없는 리팩터링 금지
3. 가정 표면화: 모호하면 코딩 전에 확인

## 커밋 규칙

- Conventional Commits: `<type>(<scope>): <description>`
- 커밋 전 반드시 사용자 확인
- Co-Authored-By: [이름] <[이메일]>
```

## 커스터마이징 가이드

사용자가 수정해야 하는 부분:
1. `사용자 환경` — 호칭, 문체, 시간대
2. `커밋 규칙` — Co-Authored-By
3. `기술 스택` — 자동 감지가 놓친 항목
4. 프로젝트 고유 규칙 추가
