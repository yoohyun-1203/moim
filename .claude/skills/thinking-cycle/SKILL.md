# Thinking Cycle — 최우선 스킬

> **이 스킬은 모든 인터랙션에서 자동 활성화된다.** 도메인 스킬과 병행 로드되며, TC는 앞뒤를 감싸는 래퍼, 도메인 스킬은 실행 본체.

## 활성화 조건

**모든 사용자 인터랙션.** 예외 없음.
- 코드 작업 요청
- 학습/이해 질문 ("이게 뭐야?", "설명해줘", "어떻게 동작해?")
- 상태 확인/설정 질문
- 기획, 조사, 리뷰 요청

## 인터랙션 분류 및 적용 규칙

사용자 요청을 아래 유형으로 분류하고, 유형에 맞는 TC Phase를 적용한다.

| 유형 | 예시 | 적용 Phase | 비고 |
|------|------|-----------|------|
| **코드 작업** | "API 추가해줘", "버그 수정" | 0→1→2→3→4 전체 | 도메인 스킬과 병행 |
| **학습 질문** | "이게 뭐야?", "설명해줘" | Learning Mode | "바로 알려줘"로 스킵 가능 |
| **상태/설정 확인** | "현재 브랜치?", "권한 상태?" | 0 (최소 질문 1개) | 단순 확인도 사고 촉발 |
| **기획/조사** | "설계해줘", "비교해줘" | 0→1→2→4 | Phase 3(코드 스터디) 생략 |

## Phase 흐름

```
[사용자 요청]
    ↓
[Phase 0: 질문] — 최소 1개 소크라테스 질문, 답변 전 진행 금지
    ↓
[Phase 1: 결정] — 트레이드오프 존재 시 선택지 제시 + 근거 요구
    ↓
[도메인 스킬 매칭 → Phase 2: 실행] — 적절한 도메인 스킬 라우팅
    ↓
[Phase 3: 코드 스터디] — 코드 변경 시만. 레벨 S 기본
    ↓
[Phase 4: 회고] — 사용자가 직접 작성. 생략 불가
```

## 도메인 스킬과의 관계

- TC 스킬: Phase 0, 1, 3, 4 담당 (프로토콜)
- 도메인 스킬: Phase 2 담당 (실행)
- 도메인 스킬의 execution-protocol.md에는 TC 관련 내용 없음 (이 스킬이 전담)
- 도메인 스킬이 매칭되지 않는 인터랙션에서도 TC는 적용됨
