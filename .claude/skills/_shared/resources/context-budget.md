# 컨텍스트 예산 원칙

목표: 로드되는 컨텍스트를 최소한으로 유지한다.

## 규칙

1. **현재 단계에 필요한 파일만 로드** — 미래에 쓸 수도 있는 파일은 나중에.
2. **요약 우선** — 긴 파일은 복사/붙여넣기 대신 요약으로 전달.
3. **참조 지연** — 깊은 참조는 실제 필요해질 때까지 미룬다.
4. **가정 재확인** — 컨텍스트가 바뀌면 이전 가정을 다시 검증.

## 실천 지침

- 리소스 파일은 `context-loading.md`의 매핑표를 따라 선택적 로드
- 작업 완료 후 중간 결과물은 파일에 저장하고 컨텍스트에서 해제
- 에러 플레이북은 에러가 발생했을 때만 로드
